# Event Manager — backend

**Описание**
- **Проект**: простой ASP.NET Core Web API для управления мероприятиями (CRUD).
- **Функции**: получение списка мероприятий, получение по id, создание, обновление и удаление.

**Требования**
- **.NET SDK**: установите .NET 10 (проверить: `dotnet --version`, ожидается `10.x`).

**Как запустить локально**
- Откройте терминал в корне репозитория и выполните:

```
cd EventManager
dotnet restore
dotnet build
dotnet run
```

- По умолчанию приложение запустится на локальных адресах (см. вывод консоли), обычно `https://localhost:5001` и `http://localhost:5000`.
- В среде разработки включён Swagger UI — откройте `/swagger` для интерактивной документации.

**API (кратко)**
- **Базовый путь**: `/events` (контроллер `EventsController`).
- **Методы**:
	- `GET /events` — получить все мероприятия
	- `GET /events/{id}` — получить мероприятие по id
	- `POST /events` — создать мероприятие (тело — `EventDTO`)
	- `PUT /events/{id}` — обновить мероприятие
	- `DELETE /events/{id}` — удалить мероприятие

**Структура репозитория**
- **Controllers**: контроллеры API
- **Services / Interfaces**: бизнес-логика и интерфейсы
- **Data**: репозиторий для хранения данных
- **Models**: DTO/модели данных

**Запуск в IDE**
- Откройте решение/проект `EventManager/EventManager.csproj` в Visual Studio или Rider, установите проект `EventManager` как стартовый и запустите (F5).
